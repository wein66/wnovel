프로젝트 이어하기: 소설 관리 사이트 (W-Novel)

1. 프로젝트 개요

목표: 스프링 부트(백엔드) + 타임리프(관리자) + 리액트 네이티브(사용자 앱) 기반 웹소설 플랫폼 구축

현재 패키지: kr.co.musicart.wnovel

DB: MariaDB

2. 현재 구현 상태 (백엔드 & 관리자)

엔티티:

User: 회원 (관리자/작가/일반), 포인트 보유

Novel: 소설 정보, 표지 이미지(URL)

Episode: 회차 정보, 본문(HTML), 1:N 이미지 리스트(EpisodeImage)

EpisodeImage: 회차별 이미지, 순서(sortOrder) 관리

기능:

관리자 로그인/로그아웃 (SecurityConfig 설정 완료)

소설 목록 조회, 등록 (표지 이미지 업로드 포함)

소설 상세 조회 (회차 목록 표시)

[진행 중] 에피소드 등록 기능 (Summernote 에디터, 이미지 다중 업로드, 제목 자동 완성 로직 구현 중)

파일 저장:

FileStoreService를 통해 로컬 uploads/novel/... 경로에 저장

WebMvcConfig에서 /novel-img/** 경로로 리소스 매핑

3. 마지막 작업 내용 & 다음 할 일

마지막 작업: EpisodeService에서 에피소드 생성 시 제목이 없으면 소설 제목을 그대로 쓰거나 이전 제목을 가져오는 로직 구현 완료.

다음 할 일:

에피소드 등록 후 실제 DB에 잘 들어가는지 테스트 및 확인

에피소드 수정/삭제 기능 구현

포인트 차감 규칙(PointRule) 관리 기능 구현

사용자 앱(React Native) 연동을 위한 REST API 구축 시작

이 상태에서 이어서 개발을 진행하자. 소스 코드는 내가 가지고 있으니 필요한 부분만 요청하면 작성해줘.
